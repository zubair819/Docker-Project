# Nginx Log Monitoring with ELK Stack (Docker)

This project demonstrates a simple centralized logging pipeline using
Nginx, Filebeat, Elasticsearch, and Kibana. The goal is to show how logs
generated by a web server can be collected, indexed, and visualized
using a containerized ELK stack.

All services are orchestrated with Docker Compose and communicate
through a shared Docker network and volume.

------------------------------------------------------------------------

## Project Overview

In this setup:

-   Nginx generates access and error logs
-   Filebeat reads and ships the logs
-   Elasticsearch stores and indexes the logs
-   Kibana is used to explore and visualize the data

This architecture reflects a common real-world observability pattern
used in modern systems.

------------------------------------------------------------------------

## Architecture Flow

Client → Nginx → Log Files → Filebeat → Elasticsearch → Kibana

### How the data moves

1.  A user sends requests to the Nginx server.
2.  Nginx writes logs to `/var/log/nginx`.
3.  A Docker volume shares these logs with Filebeat.
4.  Filebeat monitors the log files and forwards entries to
    Elasticsearch.
5.  Elasticsearch indexes the data.
6.  Kibana reads from Elasticsearch and displays the logs.

## Prerequisites

Before running the project, ensure the following tools are installed:

-   Docker (version 20.x or later recommended)
-   Docker Compose (v1.29+ or Compose v2)
-   At least 4 GB of system RAM recommended

You can verify installation with:

docker --version\
docker-compose --version

## Getting Started

### 1. Clone the repository

git clone `<your-repository-url>`{=html}\
cd `<your-project-folder>`{=html}

### 2. Build and start the services

docker compose up -d --build

This command builds the images, creates the required network and volume,
and starts all containers in the background.

### 3. Verify that containers are running

docker ps

You should see the following containers:

-   nginx
-   elasticsearch
-   kibana
-   filebeat

## Generating Sample Logs

To generate some log data:

1.  Open http://localhost in your browser.
2.  Refresh the page several times.

Each request creates entries in the Nginx access log, which Filebeat
will automatically ship to Elasticsearch.

------------------------------------------------------------------------

## Verifying Data in Elasticsearch

Run the following command:

curl -u elastic:changeme% http://localhost:9200/\_cat/indices?v

You should see an index similar to:

filebeat-7.17.0-\*

------------------------------------------------------------------------

## Viewing Logs in Kibana

Follow these steps:

1.  Open Kibana at http://localhost:5601

2.  Log in using the provided credentials

3.  Navigate to Stack Management → Index Patterns

4.  Create a new index pattern:

    filebeat-\*

5.  Choose **@timestamp** as the time field

6.  Open the Discover section

You should now be able to browse Nginx logs.

------------------------------------------------------------------------
